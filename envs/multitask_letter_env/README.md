# Multitask LetterEnv

This environment tests whether a single DDQN policy can solve a family of
LetterEnv counting tasks. Each episode samples a task identity and a count
`n`, and task progress is evaluated by an RML monitor.

The objective is to visit propositions in the task-specific order. Terms such
as `D^n` mean that the proposition `D` must be visited exactly `n` times before
the next task phase can be completed.

The `small_v1` task suite is:

| Task ID | Task |
| --- | --- |
| 0 | `A B C D^n` |
| 1 | `A B D^n C` |
| 2 | `A B C D^n C` |
| 3 | `A B D C D^n` |
| 4 | `A B C D C D^n` |

## RML Setup

The environment is RML-based. Rewards, verdicts, terminal success, and
monitor states come from task-specific RML monitors. 

Static monitor files are stored in:

- `specs/`: Prolog RML specifications, one per task.
- `configs/`: YAML monitor configuration templates, one per task.
- `configs/monitor_state_catalogue.json`: reachable RML monitor states.
- `configs/monitor_progress_catalogue.json`: RML-state progress values used for
  reward shaping.

The specs and configs are generated from the task definitions:

```bash
python -m envs.multitask_letter_env.rml_generation
```

The monitor-state catalogue is generated by replaying traces through the RML
monitors:

```bash
python -m envs.multitask_letter_env.experiments.build_monitor_state_catalogue
```

The progress catalogue is generated the same way from successful RML traces:

```bash
python -m envs.multitask_letter_env.experiments.build_monitor_progress_catalogue
```

## Observations

The observation contains:

- grid position and proposition features.
- normalized count `n`.
- one-hot task identity.
- encoded RML monitor state.

Implemented monitor encodings:

- `one_hot`: factor-template one-hot encoding derived from normalized RML
  monitor states;
- `numerical`: factor-template numerical encoding derived from normalized RML
  monitor states and their count values;
- `learned_gru`: a multitask-trained 32-dimensional GRU monitor-state encoder;
- `learned_graph`: a multitask-trained 32-dimensional graph monitor-state
  encoder.

## Testing

From the repository root, run the shared core tests, shared grid tests, and
multi-task LetterEnv tests with:

```bash
./.venv/bin/python3 -m pytest tests/core tests/letter_env_core tests/multitask_letter_env
```

The GRU and graph encoders should be trained inside this package before running
their DDQN experiments (trained models are included). Both learned encoders are trained from RML-monitor
corpora. GRU pretraining uses labelled
monitor-state traces. The labels supervise progress index, next expected event,
terminal type, and task identity.

Generate the GRU pretraining dataset:

```bash
python -m envs.multitask_letter_env.experiments.generate_gru_monitor_dataset
```

Train the GRU monitor encoder:

```bash
python -m envs.multitask_letter_env.experiments.train_gru_encoder
```

The default GRU dataset and checkpoint are written to:

```text
envs/multitask_letter_env/results_and_evaluation/encoder_pretraining/gru_dataset_small_v1_seed0/dataset.jsonl
envs/multitask_letter_env/results_and_evaluation/encoder_pretraining/gru_dim32_seed0/best_student.pt
```

The graph encoder uses RML monitor-transition traces generated from the same
task suite. Generate the graph pretraining corpus:

```bash
python -m envs.multitask_letter_env.experiments.generate_gnn_monitor_corpus
```

Train the graph monitor encoder:

```bash
python -m envs.multitask_letter_env.experiments.train_gnn_encoder
```

The default graph corpus and checkpoint are written to:

```text
envs/multitask_letter_env/results_and_evaluation/encoder_pretraining/gnn_corpus_small_v1_seed0/monitor_states.jsonl
envs/multitask_letter_env/results_and_evaluation/encoder_pretraining/gnn_small_v1_seed0/best_dynamics_encoder.pt
```

## DDQN Experiments

The main neural experiments use DDQN. The trainer starts one RML monitor per
task for training and one per task for evaluation, then trains a single
task-conditioned policy.

Example seed-0 run with the RML one-hot encoding:

```bash
python -m envs.multitask_letter_env.experiments.train_ddqn \
  --encoding one_hot \
  --seed 0 \
  --output-dir envs/multitask_letter_env/results_and_evaluation/ddqn/one_hot_n_1to5_seed0
```

The numerical encoding uses the same command with `--encoding numerical` and a
matching output directory. The default DDQN hyperparameters match the main
LetterEnv experiments: `gamma=0.9`, learning rate `0.001`, batch size `64`,
buffer size `100000`, exploration fraction `0.4`, and an RML-state progress
bonus of `10`.

After training the GRU encoder, run the learned GRU encoding with:

```bash
python -m envs.multitask_letter_env.experiments.train_ddqn \
  --encoding learned_gru \
  --seed 0 \
  --learned-gru-checkpoint envs/multitask_letter_env/results_and_evaluation/encoder_pretraining/gru_dim32_seed0/best_student.pt \
  --output-dir envs/multitask_letter_env/results_and_evaluation/ddqn/learned_gru_n_1to5_seed0
```

After training the graph encoder, run the learned graph encoding with:

```bash
python -m envs.multitask_letter_env.experiments.train_ddqn \
  --encoding learned_graph \
  --seed 0 \
  --learned-graph-checkpoint envs/multitask_letter_env/results_and_evaluation/encoder_pretraining/gnn_small_v1_seed0/best_dynamics_encoder.pt \
  --output-dir envs/multitask_letter_env/results_and_evaluation/ddqn/learned_graph_n_1to5_seed0
```

## Tabular Baselines

Tabular Q-learning baselines are included for `one_hot` and `numerical`
monitor encodings. These baselines use the same RML rewards and task suite as
the DDQN experiments, but store action values in an explicit table. They are
primarily used as a contrast between in-distribution learning on `n=1..5` and
zero-shot evaluation on larger counts.

Example seed-0 tabular run:

```bash
python -m envs.multitask_letter_env.experiments.train_tabular \
  --encoding numerical \
  --episodes 25000 \
  --seed 0 \
  --output-dir envs/multitask_letter_env/results_and_evaluation/tabular/numerical_n_1to5_seed0
```

## Zero-Shot Evaluation

Zero-shot experiments evaluate policies trained on `n=1..5` at held-out values
`n=10`, `n=15`, and `n=20`. DDQN is evaluated for all four monitor encodings.
Tabular Q-learning is evaluated for `one_hot` and `numerical`.

Example DDQN zero-shot evaluation:

```bash
python -m envs.multitask_letter_env.experiments.evaluate_zero_shot \
  --algorithm ddqn \
  --encoding numerical \
  --train-seed 0 \
  --eval-n 10 \
  --artifact-path envs/multitask_letter_env/results_and_evaluation/ddqn/numerical_n_1to5_seed0/model_final.zip \
  --output-dir envs/multitask_letter_env/results_and_evaluation/generalization/ddqn/numerical_zeroshot_n10_seed0
```

## Figures

Figures are generated from the saved CSV summaries:

- learning curves by DDQN encoding.
- zero-shot success with the tabular contrast.
- sample-efficiency bars using first success at `SR >= 0.9`.
- a tabular/neural summary table.

Generate figures with:

```bash
python -m envs.multitask_letter_env.analysis.generate_figures \
  --formats pdf png \
  --success-threshold 0.9 \
  --max-learning-steps 250000
```

## Reproducing Results

Reproduction scripts are provided in `reproduction/`. 

Run encoder pretraining:

```bash
bash envs/multitask_letter_env/reproduction/run_encoder_pretraining.sh
```

Run the main DDQN experiments for five seeds:

```bash
bash envs/multitask_letter_env/reproduction/run_ddqn_encodings.sh
```

Run tabular baselines for five seeds:

```bash
bash envs/multitask_letter_env/reproduction/run_tabular_baselines.sh
```

Run seed-0 zero-shot evaluations at `n=10`, `n=15`, and `n=20`:

```bash
bash envs/multitask_letter_env/reproduction/run_zero_shot.sh
```

Generate the report figures:

```bash
bash envs/multitask_letter_env/reproduction/run_figures.sh
```

The full selected pipeline can be run with:

```bash
bash envs/multitask_letter_env/reproduction/run_all_selected.sh
```

The full pipeline reruns encoder pretraining, five-seed DDQN experiments,
five-seed tabular baselines, seed-0 zero-shot evaluations, and figure
generation. It can take several hours depending on hardware.
